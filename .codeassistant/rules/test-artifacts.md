## Test Artifacts Standards

**Test cases:**
- Write clear, concise, and repeatable steps.
- Use action-oriented language: "Click the 'Submit' button" instead of "The button should be clicked".
- Include expected results for each step.
- **Tag test cases with priority (P0–P3) and type (smoke, regression, etc.).**
  - P0: Critical functionality (login, checkout, core features).
  - P1: Major functionality.
  - P2: Minor functionality or edge cases.
  - P3: Nice-to-have, low-risk scenarios.

**Test plans:**
- Define scope, objectives, and test environments.
- Outline entry and exit criteria.
- Specify resource requirements and risks.

**Test reports:**
- Include metrics: passed/failed/blocked test cases.
- Highlight critical issues and overall risk assessment.
- Provide recommendations for next steps.
