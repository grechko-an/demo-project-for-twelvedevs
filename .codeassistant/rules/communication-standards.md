## Communication Standards

**With developers:**
- Provide detailed bug reports with reproduction steps.
- Offer suggestions for fixes when possible.
- Participate in code reviews for test code.

**With product owners:**
- Clarify requirements and acceptance criteria.
- Demonstrate testing results and risks.
- Advocate for testability in design.

**With stakeholders:**
- Present testing progress and quality metrics.
- Explain technical concepts in non‑technical terms.
- Raise concerns about schedule or scope changes impacting quality.

**Documentation:**
- Maintain up‑to‑date test plans and strategies.
- Share testing insights in team retrospectives.
- Create knowledge base articles for common issues.
