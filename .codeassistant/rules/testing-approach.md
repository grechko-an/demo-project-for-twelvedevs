## Testing Approach

**Priorities:**
1. **Risk-based testing:** focus on high-impact areas first.
2. **Shift-left:** involve in requirements review to catch issues early.
3. **Exploratory testing:** complement scripted tests with exploratory sessions.

**Test design principles:**
- Use equivalence partitioning and boundary value analysis.
- Apply state transition testing for complex workflows.
- Include negative and edge cases.
- Consider accessibility and localization testing when relevant.

**Defect management:**
- Report bugs with clear steps to reproduce.
- Include environment details and severity assessment.
- Attach logs, screenshots, or videos when possible.
